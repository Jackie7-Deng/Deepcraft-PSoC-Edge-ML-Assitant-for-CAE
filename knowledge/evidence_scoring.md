# 证据评分规则

**更新日期**: 2026-04-14  
**定位**: 给 grounded answer、FAQ 回灌、community/case 摘要仲裁提供统一的证据优先级。

## 1. 证据等级

| 等级 | 类型 | 说明 |
|------|------|------|
| Tier 1 | 官方 PDF / datasheet / kit guide / TRM / 官方 README / 实际源码 | 主证据，关键结论优先引用 |
| Tier 2 | 本地结构化知识页 (`knowledge/*.md`) | 二级知识，负责可回答性，不覆盖 Tier 1 |
| Tier 3 | Community / KBA / Case 经验 | 经验性证据，用于补充案例与排障路径 |
| Tier 4 | `【推断/经验】` | 仅在证据不足时补充，不得伪装成规格 |

## 2. 打分建议

| 条件 | 分值 |
|------|------|
| 官方 PDF / README / 源码 | +10 |
| 含精确页码或行号 | +3 |
| 与当前平台完全一致（PSoC Edge vs PSoC 6 不混） | +3 |
| 与当前工具链/版本一致 | +2 |
| Community / Case | +5 |
| 仅为摘要、没有原始定位信息 | -3 |
| 与当前平台不明确匹配 | -4 |
| 明显依赖推断 | -6 |

## 3. 冲突处理

1. 官方 PDF / 源码冲突时，先比较版本与页码，再保持冲突显式可见。
2. Community / Case 不能覆盖官方规格；最多补充“工程上常见怎么做”。
3. `knowledge/*.md` 与原始证据冲突时，以原始证据为准，并回修知识页。

## 4. 最小输出要求

- 关键结论至少附一个 Tier 1 证据。
- 若只拿到 Tier 3 或 Tier 4，必须显式写出边界。
- 无法形成高置信结论时，优先转成“还需要哪份资料 / 哪个命令输出”。
