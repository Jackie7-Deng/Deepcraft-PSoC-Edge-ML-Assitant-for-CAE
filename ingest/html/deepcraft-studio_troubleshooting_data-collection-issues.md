# Data Collection Issues

### Other Issues

**Q:** **When recording many features or using a high sample rate, the timeline becomes slow or laggy?**

**A:** When streaming data into **DEEPCRAFT™ Studio**, the timeline can become slow or laggy if a track contains many features and/or uses a high sample rate. To improve performance during recording, click the **eye** icon next to the track name to hide the track. This disables timeline plotting for that track while the tool continues streaming and recording data in the background. Click the **eye** icon again to unhide the track and view the plotted data after recording stops. This workaround is especially useful for high‑rate streams, such as **radar**.
